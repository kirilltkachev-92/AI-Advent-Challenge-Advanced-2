# Пост-PR проверка (уровень 1 + уровень 2)

Прогони из корня репозитория, по порядку. Ни один шаг не пропускается;
падение шага не останавливает прогон — фиксируется в отчёте.

1. **Уровень 1 — код**: `cd "day 1" && JAVA_HOME=<jdk17> ./gradlew test --console=plain`.
   Запомни: сколько тестов, сколько прошло (`build/test-results/test/*.xml`,
   атрибуты tests/failures/errors).
2. **Сервис**: если healthz на 8080 не отвечает — подними `./run.sh` в фоне,
   дождись `GET /healthz` → 200.
3. **Уровень 2 — UI-smoke**: `cd "day 3/smoke" && python3 run_smoke.py --tag pr-<дата>`.
   Раннер сам делает скриншоты каждого шага и пишет `day 3/smoke-report-<tag>.md`.
4. **Если добавлялась фича, видимая в UI** — сначала обнови
   `day 3/smoke-scenarios.md` и `run_smoke.py` (новый сценарий S<n>), потом шаг 3.
5. **Единый отчёт** — `day 3/pr-check-report.md`:
   - шапка: дата, коммит (`git rev-parse --short HEAD`), что проверялось;
   - таблица «Уровень 1»: тестов / прошло / упало;
   - таблица «Уровень 2»: сценарий → PASS/FAIL (из smoke-отчёта);
   - если есть FAIL — раздел «Где проблема»: файл:строка-гипотеза по каждому
     падению (стектрейс теста или шаг сценария + скриншот);
   - вердикт: MERGE OK / НУЖНЫ ПРАВКИ.
6. Останови сервис, если поднимал его сам; иначе оставь как было.

Падения НЕ чинить в рамках этого скилла — только отчёт; фиксы уходят
профилю bug-fix отдельной задачей.
